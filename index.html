Guide 01 — Assigned to Mayo

Rename Every Layer

Almost every section in the Matthew file uses Figma's auto-generated names. The same name "Frame 602" appears on 9 different pages meaning something different each time. Claude Code cannot generate meaningful component filenames from these.

Start with the Homepage. Jorge begins the homepage build Monday — renamed Homepage layers are the #1 priority.

Naming Convention

Use this pattern: [section-purpose]-[variant]

All lowercase, hyphens between words. No spaces. No "Frame" or "Group" in the name.

You already did this correctly for three things — and it's exactly why they work well in the pipeline:

  • Main Nav Bar — clear, descriptive, used as an instance across every page
  • Footer Component — same pattern
  • Button Component - Red Filled Default — describes what it is AND its variant state

Apply this same thinking to every section on every page.

Guide 02 — Assigned to Mayo

Turn Repeated Sections into Components

The Footer and Nav Bar are proper Figma components — editing the main component updates every page. Every other repeated section is a copy-pasted frame with no shared updates.

How to Create a Component from a Frame

Find the cleanest version

For example, take the "About Division" two-column block on the Pro Truck page — it has text, image, and background.

Select the frame

Click on the section frame in the layers panel.

Create Component

Right-click → Create Component (or press Ctrl/Cmd + Alt + K).

Name it properly

Use the naming convention from Guide 1: about-division

Replace copy-pasted copies

Go to every other page that has the same section. Delete the old frame. Drag an instance of the new component from the Assets panel. Override text and images for that page.

Guide 03 — Assigned to Mayo

Add Variants & Properties

Variants are different versions of the same component that Figma treats as one thing with toggle-able options. Claude Code reads these toggles and generates code that handles all variations automatically.

Step-by-Step: Hero Component Variants

Create the first variant

Take the Homepage hero (now named hero-home) and make it a component.

Create additional variants

Duplicate the hero component. Modify the duplicate to match the division pages hero (different layout, no video, smaller height). Duplicate again for the simple hero (FAQ, Contact).

Combine as Variants

Select all hero variants. Right-click → Combine as Variants. Figma wraps them in a dashed-border Component Set.

Name the property

Figma creates "Property 1" — rename it to Type with values: Home, Division, Simple

Use it

On each page, place an instance of hero and toggle the Type property. No more copy-pasting separate frames.

Boolean Properties (Show/Hide)

For elements that sometimes appear and sometimes don't:

Select the element inside the component

For example, a subtitle text layer that only some pages use.

Add a Boolean property

In the right panel, click + next to Properties → choose Boolean → name it Show Subtitle.

Toggle per instance

On each page, turn the subtitle on or off without deleting anything.

This is what Jorge meant on the call when he talked about "a button that tells me we have one column or two columns." He was describing Variant and Boolean properties. This guide shows you how to build exactly what he needs.
Guide 04 — Assigned to Mayo — Quick Win

Set Up Design Tokens as Figma Variables

The Matthew file has no Figma Variables defined. Claude Code's token extraction tool returns empty. 30 minutes of work to fix.

Create Color Variables

Open the Local Variables panel

Click the grid icon in the right sidebar, or use the keyboard shortcut for Variables.

Create each color variable

Click Create Variable → Color for each one.

Variable NameHow to Get the ValueWhere It's Used
color/primary-redEyedrop from any CTA buttonCTAs, accents, hover states
color/background-darkEyedrop from page backgroundPage backgrounds
color/background-sectionEyedrop from alternating section bgSection backgrounds
color/text-primaryEyedrop from heading textHeadings, body text
color/text-secondaryEyedrop from subtitle/caption textSubtitles, captions
color/border-redEyedrop from red border/dividerDividers, button borders

Apply the Variables

Select a frame or text layer

Start with the Homepage hero background.

Open the color picker

Click the color swatch in the Fill or Stroke property.

Switch to Variables

Click the small hex icon → switch to the Variables tab → select the matching variable.

Priority: Colors first. Just the 6 color variables is a huge improvement. Typography and spacing can wait. This is a 30-minute task that unlocks automatic token extraction for the entire build.
